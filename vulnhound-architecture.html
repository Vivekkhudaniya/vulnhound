Stage 4
Triple-Query RAG + RRF Fusion
Three separate embedding queries against three ChromaDB collections are merged with Reciprocal Rank Fusion. Each query type captures a different angle of similarity — code structure, vulnerability pattern, and natural language semantics.
3 Query Vectors
① Code Query
Embed the raw function source code. Retrieves exploits with structurally similar Solidity.
② Pattern Query
Embed static findings + severity labels. Retrieves exploits with matching vulnerability patterns.
③ Semantic Query
Embed natural language risk description. Retrieves exploits via semantic similarity.
→
RRF Fusion
k = 60
score(d) = Σ 1/(k + rank)

Interface
CLI Commands
Full audit, static-only, and knowledge base management commands. Always prefix with PYTHONUTF8=1 on Windows.
vulnhound
# Full 7-stage audit — outputs Code4rena Markdown report
$PYTHONUTF8=1 python -m src.cli.main audit https://github.com/user/defi-protocol -o report.md
$PYTHONUTF8=1 python -m src.cli.main audit ./contracts --skip-poc -s high -o report.md

# Static analysis only (no LLM, no API key needed)
$PYTHONUTF8=1 python -m src.cli.main analyze ./contracts -t slither
$PYTHONUTF8=1 python -m src.cli.main analyze ./contracts -t all

# Knowledge base management
$PYTHONUTF8=1 python -m src.cli.main kb ingest --source defihacklabs
$PYTHONUTF8=1 python -m src.cli.main kb search "reentrancy flash loan" -k 10
$PYTHONUTF8=1 python -m src.cli.main kb stats

# Flags: --min-severity/-s --model/-m --skip-poc --verbose/-v --top-k/-k
